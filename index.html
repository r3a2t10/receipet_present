Product Motivation

Our motivation to develope our App.

收集不易

發票經常亂丟，導致要兌獎時找不到發票，造成整理上的困難。我們的應用程式不僅可以蒐集新舊式發票，更能整合到硬體端直接將資訊傳至手機，不僅整合資料、更節省紙本浪費。

繁瑣無聊

兌發票的動作重複性高，使得使用者乏味，利用遊戲化的方式解決乏味的問題，讓使用者對本應用程式的沾黏姓提升。

問卷調查

設計有關開發發票自動儲存裝置調查，以貼近人們使用上的便利性。

舊式發票影像辨識

old type receipt : number recognition.

＊Tesseract OCR (Optical Character Recognition)

＊Image Input：利用手機相機拍攝後將影像輸入。

＊Preprocessing：

            ＞進行文字區域的選擇，將影像非文字部分過濾移除雜訊。

            ＞ex:灰階後二值化

＊Segmentation：將文字影像句子分解成單一文字。

＊Feature extraction：

            ＞對每一個單一字的影像分別進行特徵值擷取，

            ＞特徵值為重要計算資訊：文字的架構、外型或像素方向。

＊Recognition：

            ＞使用取得的特徵值做分類依據，一般使用Machine Learning演算法，

            ＞不同演算法有不同辨識率


＊優點：辨識快速，英文字與數字皆有很高的準確率。

＊缺點：偶爾會有誤差，對於中文字辨識較弱


新式發票QRcode掃描

new type receipt : QRcode recognition.

＊QRCode (Quick Response Code) 快速回應碼

＊矩陣式二維條碼 用點做排列組合(1 and 0)。

＊特性：高容量、印刷面積小、易判讀、容錯率高。

＊矩陣編碼方式：電腦圖像處裡，組合編碼。

＊Zxing library：開源的1D/2D條碼圖像處理庫。

＊使用內建手機相機完成二維碼的掃描和解碼。

＊ZXing 優點：定位精準、掃描快速。

＊Zxing 缺點：QRCode字串處理、77碼。


arduino 藍芽無線傳輸技術

arduino blue tooth : data transmission

＊以arduino模擬藍芽收銀機裝置(barcode scanner＋藍芽發射器)

＊bar code scanner掃瞄商品的條碼並產生發票序號及資料

＊server端藍芽 將發票資料數值打包成多個(8 bits)位元組

＊與client端配對之後傳送發票資料到手機上


＊優點：

            ＞藍芽低功耗特性：不需要進行數據交互傳輸時可以處於「非連接」狀態(節省資源)

            ＞省去領取紙本發票的麻煩。

＊缺點：

            ＞配對的麻煩性：需要輸入驗證碼做配對才能做傳輸。

            ＞一定距離內的藍芽裝置皆可以被掃描到導致配對錯誤發票傳輸錯誤。

＊改進方法：

            ＞考慮改用NFC，短距離的高頻無線通訊技術，

            ＞可以讓裝置進行非接觸式點對點資料傳輸。
